# ESP32 Oscilloscope with ADS1115

Проект реализует простой осциллограф на базе ESP32 и 16-битного АЦП ADS1115. Данные оцифровываются с частотой 500 Гц (настраивается) и передаются по Wi-Fi через TCP-сокет (бинарный поток) и/или через USB-UART (CP2102/CH340). Коэффициент внешнего пробника (делителя) настраивается через HTTP API.

## Основные возможности

- Wi-Fi подключение, TCP-сервер на порту 9999
- Бинарная передача: timestamp (float) + напряжение (float) — 8 байт на отсчёт
- HTTP-сервер для чтения/записи коэффициента пробника (`/api/probe`)
- Дублирование данных в USB-UART (115200 бод, TX=GPIO17)
- Частота дискретизации: 500 Гц (настраивается в `config.h`)

## Требуемое оборудование

- Плата с ESP32 (ESP32-WROOM-32, ESP32-DevKitC и т.п.)
- Модуль АЦП ADS1115 (напряжение питания 3.3 В или 5 В, логические уровни совместимы с ESP32)
- Делитель напряжения (пробник) — для измерения сигналов выше ±4.096 В (например, 10:1, 100:1)
- USB-UART преобразователь (если нужен вывод на ПК) — можно использовать встроенный USB-порт, если на плате есть CP2102/CH340, но в проекте задействован UART2 (GPIO17). Для отладки можно просто смотреть TCP.

## Схема подключения

| ESP32 Pin | ADS1115 Pin | Примечание               |
|-----------|-------------|--------------------------|
| 3.3V      | VDD         | Питание модуля           |
| GND       | GND         | Общий                    |
| GPIO21    | SDA         | I2C data                 |
| GPIO22    | SCL         | I2C clock                |

**Вход АЦП:**  
ADS1115 канал AIN0 подключается к измеряемой точке относительно GND.  
Для увеличения диапазона используйте резистивный делитель на входе.  
Например, для делителя 11:1 (R1=10kΩ, R2=1kΩ) максимальное входное напряжение станет ±45 В.

## Программная среда

- [ESP-IDF](https://docs.espressif.com/projects/esp-idf/en/latest/esp32/get-started/) версии **6.0 или новее**
- Компонент `espressif/cjson` — автоматически загружается через IDF Component Manager
- Система сборки CMake

## Использование

### 1. TCP-клиент (ручной приём данных)

После подключения ESP32 к Wi-Fi запускается TCP-сервер на порту **9999**.  
При подключении клиент получает JSON-строку с форматом данных:

```json
{"format":"binary","sample_rate_hz":500,"amplitude_volts":4.096}
```

Затем сервер отправляет бинарные пакеты по 8 байт:  
- первые 4 байта – **timestamp** (float, секунды от запуска)  
- следующие 4 байта – **напряжение** (float, в вольтах с учётом коэффициента пробника)

**Пример приёма на Python:**

```python
import socket
import struct

sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
sock.connect(("IP_ESP32", 9999))
greeting = sock.recv(1024)
print(greeting.decode())

while True:
    data = sock.recv(8)
    if len(data) < 8:
        break
    ts, volt = struct.unpack('ff', data)
    print(f"{ts:.3f} s, {volt:.3f} V")
```

### 3. USB-UART вывод (опционально)

Если вы подключите USB-UART переходник к пинам **GPIO17 (TX)** и GND платы ESP32, то на скорости 115200 бод будет выводиться такой же бинарный поток (8 байт на отсчёт). Это удобно для отладки без сети.

**Примечание:** В текущей версии `main.c` функция `usb_serial_init()` не вызывается. Чтобы включить USB-вывод, добавьте в `app_main`:

```c
usb_serial_init();
```

### 4. Настройка коэффициента пробника через HTTP

По умолчанию коэффициент множителя установлен в **11.0** (соответствует делителю 11:1).  
Вы можете изменить его через HTTP-запросы.

- **Прочитать текущий множитель:**  
  `GET http://IP_ESP32/api/probe`  
  Ответ: `{"multiplier": 11.000}`

- **Установить новый множитель:**  
  `POST http://IP_ESP32/api/probe`  
  Тело: `{"multiplier": 20.0}`

Пример с `curl`:

```bash
curl -X POST http://192.168.1.100/api/probe -H "Content-Type: application/json" -d '{"multiplier":20.0}'
```

После этого все отправляемые значения напряжения будут умножены на этот коэффициент.  
Формула: `V_actual = V_ADS1115 * multiplier`, где `V_ADS1115` — напряжение, измеренное АЦП на входе AIN0 (диапазон ±4.096 В).
